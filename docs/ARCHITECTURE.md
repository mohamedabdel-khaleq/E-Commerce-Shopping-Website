


# 🏛️ بنية المشروع

يعتمد مشروع التجارة الإلكترونية هذا على بنية بسيطة من جانب العميل (Client-Side Architecture) باستخدام تقنيات الويب الأساسية (HTML, CSS, JavaScript). لا يوجد خادم خلفي مخصص لمعالجة البيانات أو المصادقة؛ بدلاً من ذلك، يتم استخدام `localStorage` لتخزين بيانات المستخدم والمنتجات وسلة التسوق والطلبات.

## 📊 المكونات الرئيسية

يتكون المشروع من عدة أقسام رئيسية، كل منها يمثل صفحة HTML وملفات JavaScript و CSS المرتبطة بها:

1.  **الصفحة الرئيسية (`index.html`)**
    *   تعرض المنتجات المميزة أو الأكثر مبيعًا.
    *   تتضمن روابط للتنقل بين أقسام الموقع.
    *   تستخدم `assets/js/top-products.js` لجلب وعرض المنتجات.

2.  **صفحات المصادقة (`auth/login.html`, `auth/signup.html`)**
    *   تتيح للمستخدمين تسجيل الدخول أو إنشاء حسابات جديدة.
    *   تستخدم `auth/auth.js` لإدارة عمليات تسجيل الدخول والتسجيل باستخدام `localStorage`.
    *   يتم تخزين بيانات المستخدم (اسم المستخدم، البريد الإلكتروني، كلمة المرور المشفرة) في `localStorage`.

3.  **صفحة المنتجات (`products/products.html`)**
    *   تعرض قائمة بجميع المنتجات المتاحة.
    *   تتضمن وظائف التصفية والبحث والفرز.
    *   تستخدم `products/products.js` لجلب المنتجات من [FakeStoreAPI](https://fakestoreapi.com/) وعرضها، بالإضافة إلى إدارة سلة التسوق.

4.  **صفحة سلة التسوق (`cart/cart.html`)**
    *   تعرض المنتجات التي أضافها المستخدم إلى سلة التسوق.
    *   تتيح للمستخدمين تعديل الكميات أو إزالة المنتجات أو إفراغ السلة.
    *   تستخدم `cart/cart.js` لإدارة سلة التسوق وعملية الدفع، وتخزين الطلبات في `localStorage`.

5.  **صفحة الطلبات (`orders/orders.html`)**
    *   تعرض سجل الطلبات للمستخدم الحالي.
    *   تستخدم `orders/orders.js` لجلب وعرض الطلبات من `localStorage`.

6.  **لوحة تحكم المسؤول (`admin/admin.html`)**
    *   (ميزة إضافية) تتيح للمسؤولين إدارة المستخدمين والمنتجات والطلبات.
    *   تستخدم `admin/admin.js` لإدارة البيانات في `localStorage`.

## 🔗 تدفق البيانات

*   **المنتجات**: يتم جلبها من [FakeStoreAPI](https://fakestoreapi.com/) عند تحميل صفحة المنتجات أو الصفحة الرئيسية.
*   **المستخدمون، سلة التسوق، الطلبات**: يتم تخزينها وإدارتها محليًا في `localStorage` الخاص بالمتصفح.
*   **المصادقة**: تتم بالكامل من جانب العميل باستخدام `localStorage`. عند تسجيل الدخول، يتم تخزين معلومات المستخدم الحالي في `localStorage`.

## 🛠️ التقنيات المستخدمة

*   **HTML5**: لهيكل الصفحات.
*   **CSS3**: للتصميم والاستجابة (Responsive Design) وتطبيق الوضع الفاتح/الداكن.
*   **JavaScript (ES6+)**: للمنطق التفاعلي، إدارة DOM، التفاعل مع `localStorage`، وإجراء طلبات API.
*   **FakeStoreAPI**: كمصدر لبيانات المنتجات.

## ⚠️ اعتبارات أمنية

نظرًا لأن المشروع يعتمد بشكل كبير على `localStorage` لتخزين البيانات الحساسة مثل معلومات المستخدم والطلبات، فإنه **غير آمن للاستخدام في بيئة إنتاج حقيقية**. `localStorage` عرضة لهجمات XSS (Cross-Site Scripting) ولا يوفر تشفيرًا للبيانات. هذا المشروع مخصص لأغراض العرض التوضيحي والتعليم فقط.
